# sdlc uninstall — Design Spec

**Date:** 2026-04-29  
**Status:** Approved

---

## Overview

Add an `uninstall` command to the `sdlc` CLI that removes all files and directories created by `init` or `update`, leaving any user-owned files untouched.

To support precise tracking, every `init` and `update` run writes a manifest file (`.agents/assets/sdlc-manifest.json`) recording every path the tool generated. `uninstall` reads this manifest and deletes only those paths.

---

## CLI Interface

Three equivalent forms, consistent with the existing `-u`/`--update` standard:

| Form | Command |
|------|---------|
| Word command | `npx sdlc uninstall` |
| Long flag | `npx sdlc --uninstall` |
| Short flag | `npx sdlc -x` |

`bin/sdlc.js` parses `-x` / `--uninstall` the same way it parses `-u` / `--update` — checking `process.argv` for the flag, then setting `SDLC_COMMAND=uninstall` and delegating to `setup.js`.

The help text lists all three forms.

## Manifest File

**Path:** `.agents/assets/sdlc-manifest.json`

Written at the end of every `init` and `update` run. Always fully replaced — never merged — so stale entries never accumulate.

```json
{
  "version": "1.0.3",
  "createdAt": "2026-04-29T10:00:00Z",
  "updatedAt": "2026-04-29T10:00:00Z",
  "platforms": ["claude", "copilot"],
  "files": [
    ".agents/skills/devops/SKILL.md",
    ".agents/skills/architect/SKILL.md",
    ".claude/CLAUDE.md",
    ".claude/QUICKSTART.md",
    ".claude/settings.json",
    ".claude/commands/sdlc.md",
    ".claude/agents/sdlc-lead.md",
    ".github/copilot-instructions.md",
    ".github/prompts/sdlc.prompt.md",
    ".github/agents/sdlc-lead.agent.md",
    "AGENTS.md"
  ],
  "gitignoreBlock": true
}
```

**Fields:**
- `version` — package version at time of last `init`/`update`
- `createdAt` — timestamp of first `init`
- `updatedAt` — timestamp of last `init`/`update`
- `platforms` — which platforms were installed
- `files` — flat list of all paths written, relative to project root. The manifest itself is **not** in this list.
- `gitignoreBlock` — `true` if the tool appended a generated block to `.gitignore`

---

## Changes to `init` and `update`

Both commands gain one new step at the end: call `writeManifest(files, platforms)`.

`writeManifest` receives the complete list of files written during the current run and writes `.agents/assets/sdlc-manifest.json`. It preserves `createdAt` from any existing manifest (first-install timestamp), updates `updatedAt` to now, and replaces `files` and `platforms` entirely.

No other logic in `init` or `update` changes.

## `uninstall` Command

### Steps

1. **Read manifest** — load `.agents/assets/sdlc-manifest.json`. If not found, print an error and exit:
   > `✗ No sdlc installation found in this project. Run npx sdlc init first.`

2. **Delete listed files** — for each path in `files[]`, delete the file if it exists. Skip silently if already gone.

3. **Remove empty parent directories** — after deletions, walk each parent directory of a deleted file. If the directory is now empty, remove it. Stop at the project root. This cleans up `.claude/commands/`, `.github/prompts/`, etc. when all their contents were sdlc-generated, but leaves `.github/` intact if it still contains user files.

4. **Clean `.gitignore`** — if `gitignoreBlock: true`, strip the block delimited by `# Generated by npx sdlc init` from `.gitignore`. Leave all other content untouched.

5. **Delete `.agents/`** — remove the entire `.agents/` directory, including the manifest itself. This is done last so the manifest is available throughout steps 1–4.

6. **Print summary** — report counts: files deleted, directories removed, and whether `.gitignore` was updated.

### Safety Invariants

- Never deletes a file not listed in the manifest.
- Never removes a directory that still contains files (user or otherwise).
- Never modifies `.gitignore` content outside the generated block.

---

## Error Handling

| Scenario | Behaviour |
|----------|-----------|
| Manifest not found | Error + exit 1 |
| Manifest is malformed JSON | Error + exit 1 |
| A listed file is already gone | Skip silently |
| `.gitignore` missing but `gitignoreBlock: true` | Skip `.gitignore` step, continue |
| `.agents/` already gone | Skip, continue |

---

## Out of Scope

- Per-platform uninstall (e.g. `npx sdlc uninstall claude`) — not needed; always removes everything.
- Dry-run flag — not in this iteration.
- Backup/restore — not in this iteration.
